# 1. 질문 변형 및 반복 실험 규약

## 1.1 질문 유형 후보

| 코드 | 유형 | 설명 | MVP 적용 |
| --- | --- | --- | --- |
| `ORIGINAL` | 기준 질문 | 최초로 제시한 질문 | 필수 |
| `SAME_QUESTION_REPEAT` | 동일 질문 반복 | 문구와 조건을 그대로 유지하여 재질문 | 필수 |
| `PARAPHRASE` | 동일 의미 표현 변형 | 판단 대상과 조건은 유지하고 문장 표현만 변경 | 선택 |
| `FRAMING_CHANGE` | 프레이밍 변화 | 장점·위험·손실·혜택 등 특정 측면을 강조 | MVP 이후 |
| `CONTEXT_CHANGE` | 맥락·조건 변화 | 정책 대안, 비용, 개인 조건 등 새로운 정보를 추가 | MVP 이후 |
| `SOCIAL_PRESSURE` | 사회적 단서 추가 | 다수 의견, 전문가 권위, 주변인의 반응 등을 제시 | MVP 이후 |
| `MODEL_CONDITION_CHANGE` | 생성 조건 변화 | 모델, temperature, 시스템 프롬프트 등을 변경 | MVP 이후 |

초기 MVP에서는 `ORIGINAL`과 `SAME_QUESTION_REPEAT`를 우선한다. 
질문 표현에 따른 강건성까지 확인할 필요가 있을 경우 `PARAPHRASE` 한 유형만 제한적으로 추가한다.

## 1.2 동일 의미 질문 인정 기준

질문을 `PARAPHRASE`로 분류하려면 다음 요소가 유지되어야 한다.

- 판단 대상
- 정책·서비스·기능의 핵심 내용
- 비용, 혜택, 적용 범위 등 주요 조건
- 행위 주체와 대상 집단
- 질문이 요구하는 응답 형태
- 에이전트에게 제공되는 정보량

다음 요소가 새롭게 추가되면 단순 표현 변형이 아니라 `FRAMING_CHANGE` 또는 `CONTEXT_CHANGE`로 분류한다.

- 새로운 정책 대안 또는 예상 결과
- 에이전트의 새로운 개인 상황
- 감정적·평가적 표현
- 전문가 권위 또는 다수 의견
- 특정 결론을 암시하는 표현

## 1.3 DDP 질문 예시

### 기준 질문

- DDP 철거에 동의하십니까? 입장과 이유를 설명해 주세요.

### 동일 의미 표현 변형 후보

- DDP를 철거하는 방안에 찬성하는지 반대하는지와 그 이유를 알려주세요.
- DDP 철거 계획에 대한 귀하의 입장과 판단 근거를 말씀해 주세요.

### 프레이밍 변화

- 유지 비용이 지속적으로 증가하고 있다는 점을 고려할 때 DDP 철거에 동의하십니까?
- 서울의 대표적인 문화 자산이라는 점을 고려할 때 DDP 철거에 동의하십니까?

### 맥락·조건 변화

- DDP를 철거하고 해당 공간에 공공주택을 건설하는 방안에 동의하십니까?
- DDP 철거 비용이 시민 세금으로 충당된다는 조건에서도 동의하십니까?

두 번째 유형부터는 새로운 정보나 평가 관점이 추가되므로 동일 질문의 반복 안정성이 아니라 조건 변화에 대한 민감도로 해석한다.

## 1.4 후속 질문 설계 원칙

반복 안정성을 평가하기 위한 후속 질문은 다음 원칙을 따른다.

- 기존 응답을 특정 방향으로 유도하지 않는다.
- 새로운 사실이나 조건을 임의로 추가하지 않는다.
- “정말 확실합니까?”처럼 압박을 주는 표현은 기본 반복 질문과 분리한다.
- 기준 질문과 가능한 한 동일한 정보량을 유지한다.
- 질문 문구, 질문 유형, 대화 이력 포함 여부를 기록한다.
- 질문 조건이 달라졌다면 동일 반복으로 처리하지 않는다.

# 2. 자동 평가 지표 후보

자동 평가는 최종 정답을 만드는 수단이 아니라, 다수의 응답에서 **변화·모순·검토 필요 사례를 우선 선별하는 수단**으로 사용한다.

## 2.1 후보 지표

| 지표 | 평가 대상 | 판정 예시 | 주요 한계 |
| --- | --- | --- | --- |
| 태도 범주 일치 | 찬성·중립·반대 유지 여부 | 찬성 → 찬성: 일치 | 세부 이유 변화는 반영하지 못함 |
| 태도 점수 변화 | 태도 강도 변화 | 4점 → 2점: 2점 감소 | 점수 추출 오류 가능 |
| 의미적 유사도 | 두 응답의 전체 의미 거리 | 임베딩 코사인 유사도 | 입장이 반대여도 단어가 비슷하면 높게 나올 수 있음 |
| NLI 모순 판정 | 두 문장의 논리적 충돌 | 함의·중립·모순 분류 | 장문, 조건부 표현, 한국어 뉘앙스에서 오판 가능 |
| LLM Judge 모순 판정 | 태도·근거의 양립 가능성 | 일치·부분 충돌·명확한 모순 | 순서·길이·모델 자기선호 편향 가능 |
| 핵심 근거 유지 | 주요 판단 이유가 유지되는지 | 비용 부담 근거 유지 | 근거 추출 자체가 주관적일 수 있음 |
| 변화 이유 분류 | 입장 변화의 설명 가능성 | 새 정보, 프레이밍, 설명 없는 변화 | 평가 기준과 프롬프트에 민감 |
| 집단 분포 변화 | 반복 실행 간 코호트 분포 | 찬성 비율 차이, 분포 거리 | 개인 안정성과 동일하지 않음 |

## 2.2 MVP 우선 지표

초기 MVP에서는 다음 항목만 우선 적용한다.

1. 태도 범주 일치 여부
2. 태도 점수 변화
3. 응답 원문 나란히 비교
4. 명확한 모순 후보 판정
5. 인간 검토 필요 여부

의미적 거리, 근거 유지율, 집단 분포 안정성은 프로토타입 결과를 본 뒤 추가한다.

## 2.3 NLI와 LLM Judge의 역할

두 문장 사이의 관계를 일반적으로 다음 세 범주로 분류한다.

- **함의:** 한 문장이 다른 문장의 내용을 지지하거나 포함함
- **중립:** 두 문장이 동시에 성립할 수 있으나 직접적인 지지 관계는 없음
- **모순:** 두 문장이 동시에 참이기 어려움

본 시스템에서는 반복 응답 사이의 명확한 태도 반전이나 사실 충돌 후보를 찾는 데 사용할 수 있다.

### LLM Judge

평가용 LLM에게 두 응답과 평가 기준을 제시하여 다음을 판정하게 한다.

- 핵심 태도가 유지되는가
- 두 응답이 동시에 양립 가능한가
- 근거가 유지되거나 합리적으로 수정되었는가
- 추가적인 인간 검토가 필요한가

LLM Judge 결과는 최종 판정이 아니라 후보 선별 결과로 저장한다. 평가 시 응답 순서를 바꾸어 재평가하고, 평가 모델과 생성 모델이 같은 경우 자기선호 가능성을 기록한다.

# 3. 인간 평가 설계 후보

## 3.1 인간 평가의 목적

사람이 합성 페르소나 역할을 직접 수행하는 것이 아니라, 자동 평가가 반복 응답을 적절히 분류했는지 확인한다.

초기에는 외부 시민을 대규모로 모집하지 않고, 팀원 또는 연구실 구성원이 일부 응답 쌍을 독립적으로 검토하는 방식이 현실적이다.

## 3.2 평가 항목 후보

| 항목 | 평가 질문 | 척도 예시 |
| --- | --- | --- |
| 태도 일관성 | 핵심 입장이 유지되는가? | 일치 / 부분 변화 / 반전 / 판단 불가 |
| 논리적 모순 | 두 응답이 동시에 성립 가능한가? | 모순 없음 / 일부 충돌 / 명확한 모순 |
| 변화의 정당성 | 입장 변화가 새 정보나 조건으로 설명되는가? | 정당화됨 / 불충분 / 설명 없음 / 해당 없음 |
| 명시적 프로필 정합성 | 응답이 나이·지역·직업 등 설정된 사실과 충돌하는가? | 충돌 없음 / 충돌 / 판단 불가 |
| 근거 관련성 | 제시한 이유가 질문과 결론에 관련되는가? | 1~5점 |
| 평가 확신도 | 평가자가 판정에 얼마나 확신하는가? | 낮음 / 중간 / 높음 |

태도나 정책 선호가 특정 프로필과 “어울리는가”를 평가하지 않는다. 
프로필 정합성은 명시적으로 설정된 사실의 충돌 여부에 한정한다.

## 3.3 소규모 평가 운영안

> 다음 수치는 확정값이 아니라 현실적인 초기 운영 예시다.
> 
- 평가자: 2명
- 평가 대상: 반복 응답 쌍 30~100개
- 표본 구성: 태도 유지, 태도 반전, 자동 지표 충돌, 소수 반응을 포함한 층화 표본
- 평가 방식: 독립 평가 후 불일치 사례 논의
- 우선 평가 항목: 태도 일관성, 모순, 변화의 정당성, 확신도

외부 참가자를 모집하지 않고 팀 내부 평가를 사용할 경우, 연구 결과에는 평가자 편향 가능성을 한계로 명시한다.

## 3.4 평가자 간 일치도 후보

| 데이터 형태 | 후보 지표 | 적용 예시 |
| --- | --- | --- |
| 평가자 2명의 범주형 판정 | Cohen’s κ | 일치·불일치, 모순 범주 |
| 순서가 있는 범주 또는 점수 | Weighted Cohen’s κ | 1~5점 근거 관련성 |
| 평가자 3명 이상 범주형 판정 | Fleiss’ κ 또는 Krippendorff’s α? | 다수 평가자 확장 시 |
| 연속형 점수 | ICC | 연속 점수를 실제로 사용할 경우 |
|  |  |  |

| 지표 | 간략한 설명 | 적합한 상황 |
| --- | --- | --- |
| **단순 일치율** | 평가자들이 동일하게 판정한 비율 | 빠른 확인용 |
| **Cohen’s κ** | 우연히 같은 답을 선택했을 가능성을 제외하고, 두 평가자의 일치도를 계산 | 평가자 2명 + 범주형 평가 |
| **Weighted Cohen’s κ** | 평가 결과의 차이에 정도가 있을 때, 가까운 불일치와 큰 불일치를 다르게 반영 | 평가자 2명 + 순서형 척도 |
| **Fleiss’ κ** | Cohen’s κ를 평가자 3명 이상으로 확장한 방식 | 여러 평가자 + 범주형 평가 |
| **Krippendorff’s α** | 평가자 수, 결측값, 범주형·순서형·연속형 등 다양한 데이터에 유연하게 적용 | 평가 구조가 복잡하거나 일부 평가가 누락된 경우 |
| **ICC** | 여러 평가자가 부여한 연속형 점수가 얼마나 유사한지 평가 | 1~5점, 0~100점 등 수치 평가 |

초기 연구에서 평가자 2명과 범주형 판정을 사용한다면 Cohen’s κ 하나만으로도 충분할 가능성이 높다.

# 4. 데이터 요소와 JSON 구조 예시

## 4.1 핵심 식별자

| 식별자 | 역할 |
| --- | --- |
| `project_id` | 연구 프로젝트 구분 |
| `cohort_id` | 코호트 조건과 추출 결과 구분 |
| `agent_id` | 원본 합성 페르소나 추적 |
| `scenario_id` | 정책·서비스·기능 시나리오 구분 |
| `question_id` | 질문과 질문 버전 구분 |
| `run_id` | 한 번의 실험 실행 구분 |
| `response_id` | 개별 생성 응답 구분 |
| `comparison_id` | 반복 응답 쌍 또는 비교 단위 구분 |

PRD 본문에서는 모든 ID를 나열할 필요가 없지만, 실제 구현에서는 에이전트·코호트·시나리오·질문·실행·응답 사이의 연결 관계가 유지되어야 한다.

## 4.4 버전 및 재현성 정보

다음 정보는 실험을 재현하고 결과를 해석하기 위해 함께 저장한다.

# 5. 상세 화면 요구사항 후보

## 5.1 실험 설정 화면

- 프로젝트명과 연구 목적
- 코호트 조건과 조건 충족 인원
- 표본 수와 seed
- 시나리오 원문 및 버전
- 질문 세트와 반복 횟수
- 모델과 생성 조건
- 예상 호출 수와 주의사항

## 5.2 실행 진행 화면

- 대상 에이전트 수
- 질문 수와 예정 응답 수
- 완료·생성 중·재시도·실패 건수
- 진행률
- 실패 유형 요약
- 선택적 재실행 기능

## 5.3 전체 결과 분석 화면

- 정상 생성과 실패 건수
- 전체 태도 및 강도 분포
- 코호트별 태도·근거·수용 조건 비교
- 시나리오 A/B/C별 반응 차이
- 주요 우려와 기대
- 소수·특이 반응 후보
- 원문 응답 목록과 필터

## 5.4 에이전트 상세 및 후속 질문 화면

- 에이전트 ID와 필요한 범위의 프로필
- 현재 및 이전 응답
- 질문·시나리오·실행 이력
- 특이 반응 후보로 선정된 이유
- 제한된 후속 질문 입력
- 후속 질문과 응답 저장

## 5.5 반복 비교 화면

- 기준 질문과 반복·변형 질문
- 질문 관계 유형
- 응답 A와 B 나란히 보기
- 태도와 점수 변화
- 자동 모순 후보 판정
- 변화 이유 요약
- 인간 검토 필요 여부

## 5.6 인간 검토 화면

- 익명화된 응답 쌍
- 질문과 필요한 최소 시나리오 정보
- 평가 항목과 척도
- 평가 확신도
- 자동 평가 결과 표시 여부 설정
- 평가자 간 불일치 사례 확인

초기 MVP에서는 별도의 인간 평가 화면을 구현하지 않고 CSV 또는 간단한 폼을 활용할 수 있다.

# 6. 선행연구 기반 설계 근거

## 6.1 합성 응답과 인간 하위집단 시뮬레이션

**Argyle et al. (2023), “Out of One, Many: Using Language Models to Simulate Human Samples.” Political Analysis.**  

https://www.cambridge.org/core/journals/political-analysis/article/out-of-one-many-using-language-models-to-simulate-human-samples/035D7C8A55B237942FB6DBAD7CAA4E49

- 사회인구학적 배경을 조건으로 언어모델 응답 분포를 인간 하위집단과 비교한 연구다.
- 본 프로젝트에는 코호트 조건과 응답 분포를 함께 기록해야 한다는 근거를 제공한다.
- 다만 특정 모델과 미국 설문 데이터에서의 결과를 한국형 합성 데이터에 그대로 일반화할 수는 없다.

## 6.2 페르소나 에이전트의 반복·내부·외부 일관성

**Kim et al. (2026), “PICon: A Multi-Turn Interrogation Framework for Evaluating Persona Agent Consistency.” arXiv preprint.**  

https://arxiv.org/abs/2603.25620

- 페르소나 일관성을 내부 일관성, 외부 사실 정합성, 반복 안정성으로 구분한다.
- 본 프로젝트의 동일 질문 반복과 모순 후보 탐색에 직접적인 참고가 된다.
- 현재 사전 공개 논문이므로 후속 출판 및 평가 결과를 확인할 필요가 있다.

## 6.3 인간 대화 기준을 활용한 페르소나 시뮬레이션 평가

**Bao et al. (2026), “Eval4Sim: An Evaluation Framework for Persona Simulation.” arXiv preprint.**  

https://arxiv.org/abs/2603.02876

- 페르소나 시뮬레이션을 adherence, consistency, naturalness의 여러 차원으로 평가한다.
- 하나의 불투명한 종합 점수보다 평가 축을 분리해야 한다는 근거로 참고할 수 있다.
- 본 프로젝트의 초기 MVP에서는 모든 평가 축을 구현하지 않고 반복 안정성과 명시적 사실 충돌을 우선한다.

## 6.4 LLM Judge와 편향 통제

**Zheng et al. (2023), “Judging LLM-as-a-Judge with MT-Bench and Chatbot Arena.”**  

https://arxiv.org/abs/2306.05685

- LLM을 개방형 응답 평가자로 활용할 가능성과 함께 순서, 장황성, 자기선호 등의 편향을 다룬다.
- 본 프로젝트에서는 LLM Judge를 최종 정답이 아니라 검토 후보를 선별하는 보조 수단으로 사용한다.

## 6.5 자연어 추론 기반 모순 판정

**Williams, Nangia, and Bowman (2018), “A Broad-Coverage Challenge Corpus for Sentence Understanding through Inference.” NAACL.**  

https://aclanthology.org/N18-1101/

- 자연어 추론에서 문장 관계를 함의, 중립, 모순으로 분류하는 대표적인 기반을 제공한다.
- 본 프로젝트에서는 반복 응답 간 모순 후보 탐색에 활용할 수 있지만, 장문·조건부·한국어 응답에 대한 별도 검증이 필요하다.

## 6.6 근거 품질의 다차원 평가

**Wachsmuth et al. (2017), “Computational Argumentation Quality Assessment in Natural Language.” EACL.**  

https://aclanthology.org/E17-1017/

- 논증 품질을 단일 점수가 아니라 여러 하위 차원으로 구분한다.
- 본 프로젝트에서는 근거의 관련성·수용 가능성·충분성을 응답 안정성과 별도로 볼 수 있다는 근거로 참고한다.
- 근거 품질 평가는 초기 MVP의 필수 범위가 아니라 후속 확장 후보다.
