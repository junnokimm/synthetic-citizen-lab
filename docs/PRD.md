# 1. 프로젝트 개요

## 1.1 프로젝트 한 줄 정의

한국형 합성 페르소나 데이터에서 연구 목적에 맞는 에이전트 코호트를 구성하고,  
정책·서비스·기능에 대한 반응을 생성한 뒤,  
**전체 및 코호트별 반응, 소수·특이 반응, 특정 에이전트의 후속 응답, 기본적인 반복 안정성**  
을 탐색할 수 있도록 지원하는 모듈형 가상 실험 환경

## 1.2 핵심 목적

본 프로젝트는 단순히 합성 페르소나에게 질문하고 응답을 생성하는 데 그치지 않고,  
다음 과정을 하나의 연구 흐름으로 제공하는 것을 목표로 한다.

- 연구 목적에 맞는 페르소나 집단 구성
- 정책·서비스·기능에 대한 에이전트별 반응 생성
- 전체 및 코호트별 반응 차이 탐색
- 전체 요약에서 사라질 수 있는 소수·특이 반응 확인
- 특정 에이전트의 반응 원인과 수용 조건에 대한 후속 질문
- 동일 질문 반복을 통한 기본적인 응답 안정성 확인

## 1.3 기본 해석 범위

본 시스템은 실제 한국 시민의 여론이나 정책 수용률을 예측하는 시스템이 아니다.  
결과는 사용한 **합성 데이터, 생성 모델, 프롬프트, 질문 및 코호트 조건 안에서 나타난 반응 패턴**으로 한정하여 해석한다.

# 2. 프로젝트 배경 및 문제 정의

대규모 합성 페르소나 데이터가 존재하더라도, 이를 실제 연구에 활용하기 위해서는 연구자가 대상 선정, 질문 설계, 응답 생성, 결과 분석과 반복 검증 과정을 별도로 구현해야 한다.

현재 단순 응답 생성 방식에는 다음과 같은 한계가 있다.

1. **재현 가능한 실험 대상 구성의 어려움**

   표본 추출 조건과 seed가 기록되지 않으면 동일한 에이전트 집단을 다시 구성하기 어렵다.

2. **실험 조건과 응답 추적의 어려움**

   어떤 시나리오, 질문, 모델 조건에서 특정 응답이 생성되었는지 사후에 확인하기 어렵다.

3. **다수 응답의 비교와 소수 의견 탐색의 어려움**

   전체 응답을 단순 요약하면 코호트별 차이와 낮은 빈도의 의견이 사라질 수 있다.

4. **특정 에이전트 후속 탐색의 어려움**

   다른 반응을 보인 에이전트를 발견해도 해당 에이전트의 이전 응답과 연결하여 후속 질문하기 어렵다.

5. **응답 변화 해석의 어려움**

   동일 질문 반복에서 발생한 불안정과 새로운 정보·조건에 따른 합리적인 변화를 구분하기 어렵다.

따라서 데이터 조회부터 코호트 구성, 반응 생성, 결과 탐색, 후속 질문과 기본 검증까지 연결된 연구 환경이 필요하다.

# 3. 제품 목표 및 연구 질문

## 3.1 제품 목표

본 시스템은 연구자가 다음 작업을 수행할 수 있도록 한다.

- 조건과 seed를 이용해 재현 가능한 코호트를 구성한다.
- 시나리오와 질문을 설정하고 에이전트별 응답을 생성한다.
- 응답 원문과 구조화된 태도·근거를 함께 저장한다.
- 전체, 코호트별, 시나리오별 반응 차이를 비교한다.
- 소수·특이 반응 후보와 해당 에이전트의 원문을 확인한다.
- 특정 에이전트에게 구조화된 후속 질문을 진행한다.
- 동일 질문 반복 결과를 비교해 기본적인 안정성을 확인한다.
- 실험 조건과 결과를 저장하고 내보낸다.

## 3.2 초기 연구 질문

### RQ1. 전체 및 코호트별 반응

특정 정책, 서비스 또는 기능을 제시했을 때 어떤 태도, 우려, 기대, 수용 조건이 나타나며, 이러한 반응은 코호트별로 어떻게 다르게 생성되는가?

### RQ2. 시나리오 및 대안별 반응 차이

정책안·서비스안·기능안 A/B/C 또는 기능 추가 전후에 전체 및 코호트별 반응은 어떻게 달라지는가?

### RQ3. 소수, 특이 반응

전체 반응에서 빈도가 낮거나 다른 반응과 구별되는 의견은 무엇이며, 해당 에이전트에게 후속 질문했을 때 어떤 추가 이유와 수용 조건이 나타나는가?

### RQ4. 기본 반복 안정성

동일한 에이전트, 시나리오, 질문 및 모델 조건에서 반복 실행했을 때 핵심 태도와 근거는 어느 정도 유지되는가?

(초기 연구에서는 반응 탐색을 중심에 두고, 반복 안정성 검증은 결과를 해석하기 위한 기본 검증 절차로 활용)

# 4. 주요 사용자 및 활용 시나리오

## 4.1 주요 사용자

- 합성 페르소나 및 에이전트 기반 연구자
- HCI 및 Human-AI Interaction 연구자
- UX 리서처
- 사회과학 연구자
- 정책·공공서비스 및 제품 기획 연구자

초기 MVP는 일반 사용자가 아니라, 실험 조건과 결과의 한계를 이해하고 연구 목적으로 사용하는 사용자를 대상으로 한다.

## 4.2 시나리오 A: 정책 또는 공공서비스 반응 탐색

연구자가 정책 설명과 코호트 조건을 입력하면 시스템은 전체 및 코호트별 태도, 주요 우려, 기대와 수용 조건을 보여준다.

**예상 결과**

- 전체 태도 분포
- 코호트별 태도 및 근거 차이
- 주요 우려와 기대
- 정책 수용에 필요한 조건
- 소수·특이 반응 후보

## 4.3 시나리오 B: 서비스 콘셉트 및 신규 기능 비교

기존 서비스안과 신규 기능안 또는 기능 A/B/C를 동일한 코호트와 질문으로 비교한다

**예상 결과**

- 대안별 사용 의향 및 태도 차이
- 기능 추가 전후의 우려와 기대 변화
- 코호트별 선호 기능과 수용 장벽
- 실제 사용자 조사에서 확인할 가설과 후속 질문 후보

## 4.4 시나리오 C: 소수, 특이 반응 후속 탐색

전체 반응과 다르거나 낮은 빈도의 의견을 보인 에이전트를 선택하여,  
왜 해당 반응을 보였는지와 어떤 조건에서 입장이 달라지는지를 확인한다.

초기 MVP에서는 자유로운 장기 대화보다  
**미리 정의된 후속 질문 또는 연구자가 입력한 제한된 후속 질문**을 우선.

# 5. 프로젝트 범위 및 비목표

- 대규모 합성 페르소나 데이터 조건 조회

- 시나리오 및 질문 설정
- 에이전트별 반응 생성과 저장

- 소수·특이 반응 후보 탐색
- 특정 에이전트 상세 조회와 후속 질문
- 동일 질문 반복 비교
- 결과 내보내기

- 실제 한국 시민의 여론이나 정책 수용률 예측
- 합성 응답을 실제 인간 응답으로 간주
- 합성 데이터의 인구 대표성 주장
- 결과만을 이용한 실제 의사결정 자동화
- 원본 전체 에이전트에 대한 대규모 LLM 전수 호출
- 초기 MVP에서의 에이전트 간 토론 및 사회적 확산 시뮬레이션
- 자유형 장기 멀티턴 인터뷰 시스템
- 복잡한 자동 평가 지표와 다중 모델 평가 체계의 완전한 구현
- 실제 시민 응답과의 대규모 비교 실험

# 7. MVP 범위

## 7.1 MVP 핵심 흐름

> **코호트 구성 → 반응 생성 → 전체·코호트별 비교 → 특정 에이전트 조회 → 동일 질문 반복 비교**

## 7.2 MVP 필수 기능

- Parquet·DuckDB 기반 조건 조회
- 조건과 seed 기반 코호트 추출
- 시나리오 및 질문 직접 입력
- 에이전트별 원문 및 구조화 응답 생성
- 실행 조건과 응답 저장
- 전체 및 코호트별 태도·근거 비교
- 빈도 기반 소수 반응 후보 확인
- 특정 에이전트 응답 이력 조회
- 제한된 후속 질문 기능
- 동일 질문 2~3회 반복 비교
- CSV 또는 JSON 내보내기

## 7.3 MVP 이후 검토 기능

- 질문 자동 패러프레이즈 생성
- 프레이밍·맥락 변화 실험 자동화

# 8. 핵심 기능 요구사항

## FR-01. 데이터 조회 및 코호트 구성

**목적**

원본 데이터를 수정하거나 전체 메모리에 적재하지 않고 조건에 맞는 에이전트를 조회하고 재현 가능한 표본을 구성한다.

**핵심 기능**

- 주요 컬럼과 기본 분포 확인
- 인구통계·사회경제·생활·성격 조건 필터
- 조건 일치 인원 수 확인
- 표본 수와 seed 기반 추출
- 코호트 조건과 에이전트 ID 저장

**MVP 완료 기준**

동일한 원본 데이터, 조건, 표본 수와 seed에서 동일한 에이전트 ID 목록을 다시 구성할 수 있다.

## FR-02. 시나리오 및 질문 설정

**목적**

정책·서비스·기능 설명과 연구 질문을 실험 단위로 관리한다.

**핵심 기능**

- 시나리오 텍스트 직접 입력
- 정책안·기능안 A/B/C 등록
- 질문 세트 작성 및 수정
- 사용한 시나리오와 질문 버전 저장

**MVP 완료 기준**

각 응답이 어떤 시나리오와 질문을 기반으로 생성되었는지 확인할 수 있다.

## FR-03. 에이전트 반응 생성 및 저장

**목적**

페르소나 프로필, 시나리오와 질문을 결합하여 에이전트별 반응을 생성하고 추적 가능하게 저장한다.

**핵심 기능**

- 필요한 프로필 정보 조회
- 자유 서술형 응답 생성
- 태도, 강도, 핵심 근거 등 최소 구조화 정보 생성
- 모델·프롬프트·반복 회차 저장
- 성공·실패 건수 기록

**MVP 완료 기준**

특정 응답이 어떤 에이전트와 실험 조건에서 생성되었는지 추적할 수 있다.

## FR-04. 전체·코호트·대안별 반응 분석

**목적**

다수 응답에서 전체적인 반응과 코호트 및 대안별 차이를 탐색한다.

**핵심 기능**

- 전체 태도 분포
- 코호트별 태도와 주요 근거 비교
- 시나리오 A/B/C 반응 비교
- 주요 우려, 기대와 수용 조건 정리
- 원문 응답 확인

**MVP 완료 기준**

사용자가 전체 결과뿐 아니라 선택한 코호트와 시나리오별 반응 차이를 확인할 수 있다.

## FR-05. 소수·특이 반응 및 에이전트 후속 탐색

**목적**

전체 요약에서 사라질 수 있는 반응을 확인하고 특정 에이전트의 응답 배경을 추가로 탐색한다.

**핵심 기능**

- 낮은 빈도의 태도, 근거 확인
- 반복 실행에서 태도가 변한 에이전트 확인
- 에이전트 프로필과 원문 응답 조회
- 제한된 후속 질문과 응답 이력 저장

**MVP 완료 기준**

사용자가 후보 선정 이유와 원문을 확인하고 해당 에이전트에게 후속 질문할 수 있다.

## FR-06. 반복 검증 및 결과 내보내기

**목적**

동일 조건에서 응답의 기본 안정성을 확인하고 실험 결과를 외부에서 재분석할 수 있도록 한다.

**핵심 기능**

- 동일 질문 반복 실행
- 태도 범주 및 점수 변화 확인
- 반복 응답 원문 나란히 비교
- 명확한 태도 반전 후보 표시
- CSV·JSON 결과 내보내기

**MVP 완료 기준**

에이전트별 반복 응답의 태도와 원문 차이를 비교하고 실험 조건과 함께 내보낼 수 있다.

# 9. 반응 분석 및 기본 검증 방식

## 9.1 반응 분석

생성된 응답은 다음 수준에서 탐색한다.

- **전체 반응:** 태도, 주요 기대, 우려와 수용 조건
- **코호트별 반응:** 연령, 지역, 직업, 소득 등 조건별 차이
- **대안별 반응:** 정책안·서비스안·기능안 A/B/C 차이
- **소수 반응:** 빈도가 낮거나 전체 요약에서 드러나지 않는 의견

코호트별 차이는 실제 집단의 특성이나 인과관계로 해석하지 않고, 합성 데이터와 생성 조건 안에서 나타난 패턴으로 제한한다.

## 9.2 기본 반복 검증

MVP에서는 다음 정도의 검증을 우선한다.

1. 동일한 에이전트와 질문을 2~3회 반복한다.
2. 태도 방향과 태도 점수의 변화를 확인한다.
3. 핵심 근거가 유지되는지 원문을 나란히 비교한다.
4. 찬성에서 반대와 같이 명확히 뒤집힌 사례를 검토 대상으로 표시한다.
5. 중요한 사례는 사람이 직접 원문을 확인한다.

(동일 의미 표현 변형, 프레이밍 변화, NLI, LLM Judge 기반 모순 판정 등은 초기 MVP 이후 단계에서 구체화)

## 9.3 사람 평가의 범위

초기 단계에서는 실제 시민을 대규모로 모집하여 합성 응답과 비교하지 않는다.

필요한 경우 팀원 또는 연구실 구성원 2명 정도가 소규모 응답 표본을 독립적으로 검토하여 다음을 확인한다.

- 태도 유지 여부
- 명확한 모순 여부
- 프로필에 명시된 사실과의 충돌 여부

구체적인 평가자 수와 표본 크기는 미팅 이후 연구 설계 단계에서 확정한다.

# 10. 데이터 및 기술 구조

## 10.1 저장할 핵심 정보

- 원본 데이터 버전 또는 fingerprint
- 코호트 조건, seed와 에이전트 ID
- 시나리오 및 질문 버전
- 모델과 프롬프트 조건
- 응답 원문
- 태도, 강도와 핵심 근거
- 반복 회차
- 생성 성공·실패 정보
- 후속 질문 이력

## 10.2 기술 구조

```text
합성 페르소나 Parquet
  → DuckDB 조건 조회
  → seed 기반 코호트 구성
  → 시나리오·질문 설정
  → 에이전트 반응 생성
  → 결과 저장
  → 전체·코호트·대안별 분석
  → 에이전트 후속 탐색 및 반복 비교
```

# 11. 성공 기준 및 연구 윤리

## 11.1 제품 성공 기준

- 전체 데이터를 메모리에 적재하지 않고 조건 조회가 가능하다.
- 동일 조건과 seed로 동일한 코호트를 다시 구성할 수 있다.
- 모든 응답이 에이전트, 시나리오, 질문과 모델 조건에 연결된다.
- 전체·코호트·대안별 반응 차이를 확인할 수 있다.
- 소수 반응 후보의 원문과 선정 이유를 확인할 수 있다.
- 특정 에이전트의 응답과 제한된 후속 질문 이력을 조회할 수 있다.
- 동일 질문 반복 결과를 비교할 수 있다.
- 실험 조건과 결과를 함께 내보낼 수 있다.

## 11.2 연구 지원 성공 기준

본 프로젝트의 성공은 합성 에이전트가 실제 사람과 동일하거나 항상 일관되게 답하는 것을 증명하는 데 있지 않다.

다음이 가능하면 초기 연구 도구로서 의미가 있다.

- 정책·서비스·기능에 대한 반응 유형을 탐색할 수 있다.
- 코호트와 대안별 차이를 비교할 수 있다.
- 전체 요약에서 사라질 수 있는 반응을 확인할 수 있다.
- 기본 반복 검증으로 결과의 변동성을 확인할 수 있다.
- 결과의 생성 조건과 한계를 함께 기록할 수 있다.

## 11.3 연구 윤리 및 해석 원칙

- 합성 응답을 실제 인간 응답으로 제시하지 않는다.
- 코호트별 차이를 실제 인구집단의 특성이나 인과관계로 일반화하지 않는다.
- 인구통계 속성만으로 특정 태도를 가져야 한다고 판단하지 않는다.
- 이름, 상세 주소 등 직접 식별 가능성이 있는 정보는 기본적으로 노출하지 않는다.
- 원본 데이터 전체를 Git에 포함하지 않는다.
- 자동 분석 결과와 함께 원문 응답을 확인할 수 있게 한다.
